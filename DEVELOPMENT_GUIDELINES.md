# 🚀 자동 가격 조정 웹앱 - 개발 지침서 (Development Guidelines)

이 문서는 인공지능(AI)과 개발자가 협업할 때 반드시 지켜야 할 핵심 규칙과 아키텍처 원칙을 정의합니다.
**AI는 향후 코드를 수정하거나 새로운 기능을 제안할 때, 항상 이 문서를 최우선으로 읽고 원칙을 반영하여야 합니다.**

## 1. 기본 원칙 (Core Principles)
* **언어 및 주석:** 코드 내 핵심 주석, 출력되는 가이드 문서 표기, 그리고 에러 메시지들은 항상 사용자가 이해하기 쉬운 **'한국어'**를 기본으로 작성합니다.
* **안정성 최우선:** 결제, 가격 조정 연산(Price Engine), 마켓 API 연동 등 돈과 관련된 핵심 로직은 반드시 보수적인 에러(예외) 처리 원칙을 따릅니다.
* **점진적 개발:** 여러 파일의 구조를 한 번에 크게 바꾸기보다는, 하나의 작은 기능 단위가 사이드 이펙트 없이 완벽히 동작하는지 테스트한 후 다음 개발로 넘어갑니다.

## 2. 코드 스타일 & 기술 스택 (Tech Stack & Style)
* **프론트엔드 환경:** React, Vite 환경에서 TypeScript를 적극 활용하여 타입 안정성을 확보합니다. 
* **UI/UX 디자인:** 
  * 현재 훌륭하게 구축된 다크 모드 기반의 프리미엄(Premium) UI 일관성을 해치지 않습니다. Tailwind CSS와 shadcn/ui 기반 컴포넌트를 우선적으로 재사용합니다.
  * 백그라운드 색상 지정 실수로 인해 텍스트 가독성이 떨어지지 않도록 다크 모드의 텍스트 대비(Contrast)를 철저히 검사합니다.

## 3. 가격 수정 로직 특별 원칙 (Price Engine Rules)
* **최저 한계선 (마진 방어):** 어떠한 오류나 극단적인 치킨 게임 상황에서도, 사용자가 설정한 **[최소 보장 마진율]** 이하로 가격이 하락하도록 API 요청을 보내지 않습니다.
* **API 호출량 엄수:** 오픈마켓(네이버, 쿠팡 등)의 API 호출 횟수 제한(Rate Limit) 정책을 넘어 차단되지 않도록 크롤러와 백그라운드 워커의 실행 주기(Cron)를 적절히 조절합니다.

---
## 4. 앱 배포 및 데이터 관리 (PWA & Offline Data)
* **PWA (Progressive Web App) 우선 적용:** 웹 기반으로 개발하되, 사용자가 PC 바탕화면이나 모바일에 아이콘을 생성해 네이티브 앱(Desktop App)처럼 단독 실행할 수 있도록 PWA 환경을 필수로 구성합니다.
* **데이터 내보내기/가져오기 (JSON):** 클라우드 외에도 사용자 로컬 PC에 직접 오프라인 백업을 할 수 있도록, 설정값과 DB 데이터를 `JSON/CSV` 파일 형태로 Export/Import 하는 기능을 구현합니다.
* **업데이트 자동화:** PWA 특성을 살려 서버(Vercel) 코드가 수정되면 사용자가 다음 접속 시 자동으로 최신 버전을 캐싱하여 내려받도록 서비스 워커(Service Worker)를 구성합니다.

## 5. AI 시니어 파트너 특별 준수 사항 (Project Management)
1. **업무 시작 전 준수 사항:** 복잡한 요청에 대해 바로 실행하지 말고, 먼저 **[작업 계획서]**를 작성하여 승인을 받습니다. 관련된 매뉴얼이나 가이드라인을 사전 숙지합니다.
2. **작업 중 기억 유지 (Memory Management):** 작업의 흐름을 놓치지 않기 위해 루프 프로젝트 최상단에 `plan.md`(전체 설계도), `context.md`(주요 결정 사항 및 배경), `todo.md`(완료 및 남은 항목 체크리스트) 파일을 실시간으로 관리합니다. 매번 답변 마지막에는 현재 `todo.md`의 진행 상황을 요약합니다.
3. **단계별 실행 원칙:** 한 번의 답변에 1~2개 항목씩만 끊어서 진행하고 사용자의 피드백을 받습니다. 
4. **품질 관리 및 보고 (Self-Review):** 작업 완료 후 규칙 준수 여부와 보안/에러 허점을 점검하며, 답변 마지막에 **[셀프 체크 리포트]**를 작성해 보고합니다.
5. **페르소나 설정:** 단순한 명령 수행 챗봇이 아닌, 능동적으로 품질을 책임지는 **'시니어 파트너'**로 행동합니다.
